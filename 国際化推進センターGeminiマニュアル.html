はじめに

このマニュアルは、山梨大学グローバル推進課の職員の皆さんが、日々の業務において Google Workspace for Education 環境で提供される Gemini を安全かつ効果的に活用するための一助となることを目的としています。Geminiは文章作成、翻訳、情報収集、アイデア出しなど、多岐にわたる業務の効率化に貢献できる可能性を秘めています。

1. 前提：Geminiを安全・効果的に使うために

1.1 Google Workspace for Education 版 Gemini の利用とデータ保護

  • データ保護について (確認中): Google Workspace for Education 環境では、原則として Gemini への入力内容がモデルの学習に使用されないとされています。現在、大学として契約内容の詳細な影響について最終確認を行っております。確認が完了するまでは、特に機密性の高い情報の入力は極力控えていただくようお願いいたします。 正式な案内があり次第、改めて周知します。

  • 【最重要・厳守事項】 上記の状況に関わらず、機密情報や個人情報を自由に入力して良いという意味では決してありません。 業務で取り扱う情報には機密性の高いものが含まれます。個人情報（学生、教職員、その他関係者の氏名、連絡先、学籍番号、成績、健康情報など）や、学外秘とされる情報（非公開の協定内容、研究情報など）を入力することは絶対に禁止です。 必ず学内の情報セキュリティポリシーおよび関連規定を遵守してください。

  • 指定アカウントでの利用: 必ず西崎先生から付与されたGoogleアカウント（****@alps-lab.org）でログインしてGeminiを利用してください。個人アカウントや他のアカウントとはデータ保護のレベルや利用規約が異なります。
  • ブラウザの分離（推奨）: 誤操作防止やセキュリティの観点から、普段の業務で使用しているブラウザとは別に、Gemini専用のブラウザプロファイルを作成することを推奨します。（例: Chromeであればプロファイルを追加する機能を利用）

1.2 プロンプトの重要性と改善プロセス

  • プロンプトが鍵: Geminiから質の高い回答を得るためには、明確で具体的な指示（プロンプト）が不可欠です。「何を」「どのように」してほしいのかを丁寧に伝えることが重要です。
  • 対話による改善: 最初から完璧な出力が得られるとは限りません。期待通りでない場合は、Geminiと対話を繰り返し、指示を追加・修正していくことで、満足のいく結果に近づけることができます。試行錯誤を恐れず、何度もやり取りしてみましょう。
  • 再現性の確保: 良い出力が得られたら、どのようなプロンプトでその結果に至ったかを記録しておくことが重要です。後述するMARKDOWN形式でプロンプトを構造化してメモしておくと、後で再利用しやすくなります。

1.3 【重要】Gemの活用（カスタム指示）

  • Gemとは: Geminiには「Gem」と呼ばれるカスタム指示機能があります。これは、よく使う指示や前提条件を事前に登録しておくことで、毎回プロンプトに入力する手間を省くための機能です。

  • 利用可否の確認: Gem（カスタム指示）機能が利用可能かどうかは、契約している Google Workspace for Education のプランによって異なる場合があります。利用できない場合は、プロンプト内に都度指示を記述してください。

  • 登録メリット: (利用可能な場合) グローバル推進課の業務で頻繁に使うであろう「役割設定」や「文体・表現の指定」などを登録しておくと、作業効率が格段に向上します。

2.2 英語メールの作成支援

  • 目的: ゼロから英語メールを作成する負担を軽減し、自然で適切な表現のドラフト（下書き）を効率的に得ることを目指します。完成した日本語メールを丸ごと翻訳させるよりも、意図が伝わりやすく、質の高い英文が得られやすいです。
  • 手順:
    1. 伝えたい内容、メールの要点を日本語で箇条書きにします。（誰に、何を伝えたいのか、何をしてほしいのか等）
    2. メールの目的、相手との関係性に応じたトーン（丁寧さ）を指定します。
    3. 「日本語母語話者として適切な英語表現で」といった指示を加えることで、不自然にこなれすぎた表現を避けることができます。（この指示はGemに登録推奨、利用可能な場合）
    4. 以下のMARKDOWN形式プロンプトを参考に、Geminiにドラフト作成を依頼します。
    5. 生成されたドラフトを確認し、内容の正確性、ニュアンス、表現をチェックし、必要に応じて修正します。Geminiに修正指示を出して改善することも可能です。

  • 推奨モデルレベル: 標準 (Pro相当推奨)
    自然で適切な表現や、指定された条件（トーン、CEFRレベル等）を考慮した生成には、Pro相当のモデルが推奨されます。（利用可能な最上位モデルはプランによります）

  • プロンプト例 (MARKDOWN): 
    # 英語メール作成支援 (下書き依頼)

## 指示
以下の要点と条件に基づき、英語でメールの下書きを作成してください。

## 条件
- 宛先: [相手の名前や所属、関係性など。例: 〇〇大学 交換留学担当 Prof. Smith 宛]
- 差出人: [自分の所属と名前。例: 山梨大学 グローバル推進課 田中一郎]
- 目的: [メールを送る目的を明確に。例: 交換留学プログラムに関する問い合わせ]
- トーン: [希望する丁寧さの度合い。例: 非常に丁寧 (Very Formal), 標準的なビジネスメール (Standard Business)]
- 英語表現レベル: 日本語母語話者が書いたような、過度にこなれていない自然な表現 (CEFR B2レベル相当) を使用してください。(※Gemが利用可能で登録済みの場合は省略可)

## 伝えたい要点 (日本語で箇条書き)
- [例1: 先日お送りした協定更新に関する書類はお受け取りいただけましたでしょうか？]
- [例2: 受け入れ留学生リストについて、確認したい点が2点あります。]
- [例3: 次回のオンラインミーティングの日程調整をお願いできますでしょうか？]
- [例4: 添付の資料をご確認ください。]

## その他
- 適切な件名 (Subject) も提案してください。
- 結びの言葉 (Closing) も含めてください。

2.3 CNS掲示（学内掲示板）の英語版作成

  • 目的: 日本語で作成された学内掲示の内容を、留学生を含む多様な構成員に正確かつ分かりやすく伝えるための英語版を作成することを目指します。

    1. 元の日本語掲示のテキスト全体をコピーします。

    3. 読者には英語を母語としない人も多く含まれることを考慮し、「CEFR B2レベルまでの表現で」のように、平易な言葉遣いを指定することが有効です。

  • 推奨モデルレベル: 標準 (Pro相当推奨)
    翻訳の正確性や、CEFRレベルといった細かい指示への対応を考慮すると、Pro相当のモデルが推奨されます。（利用可能な最上位モデルはプランによります）

2.4 申請書作成の補助

  • 目的: 提出が必要な申請書について、関連資料を基に設問への回答案を作成する作業を補助します。

  • 【最重要・厳重注意】 申請書には極めて機密性の高い情報や個人情報が含まれる可能性が非常に高いです。Geminiに入力する情報には最大限の注意を払い、原則として個人情報や機密情報は入力せず、マスキング・匿名化を徹底してください。 特に学外機関への申請など、機密保持が厳格に求められる場合は、Gemini利用の可否及び利用方法について、必ず事前に上司や関係部署に確認し、指示に従ってください。 安易な利用は重大な情報漏洩インシデントにつながるリスクがあります。利用は慎重に判断してください。

  • 手順:
    1. 回答の根拠となる資料を用意します。（ファイルアップロード機能が利用可能で、かつ機密情報に配慮できる場合に活用。または、関連情報をテキスト入力。機密性・個人情報に最大限配慮）
    2. 申請書の設問項目をコピー＆ペーストします。（設問自体に機密情報が含まれていないか確認）
    3. 各設問に対し、どのような方向性で回答したいか、盛り込みたい要素などを指示します。（指示内容にも機密情報を含まないように注意）
    4. 希望する記述スタイル（箇条書き、説明文形式など）や詳細度を指定します。
    5. 以下のMARKDOWN形式プロンプトを参考に、Geminiに回答案作成を依頼します。
    6. 【厳重確認】 生成された回答案を必ず、細心の注意を払って確認します。事実関係の正確性、表現の適切さはもちろん、意図せず機密情報や個人情報が生成・混入していないかを厳しくチェックし、責任をもって修正・編集してください。

  • 推奨モデルレベル: 高性能 (Pro相当以上推奨)
    長文の根拠資料読解（可能な場合）、複雑な指示の理解、質の高い文章生成が求められるため、Pro相当以上の高性能モデルの利用が強く推奨されます。（利用可能な最上位モデルはプランによります）

## スライド構成案作成 (MARP形式)
- 上記の概要と収集情報に基づき、タイトルスライドを含めて[おおよその枚数、例: 8枚]程度のスライド構成案を**MARP形式**で作成してください。
- 各スライドには、タイトル (`#` や `##` を使用) と、主要なコンテンツ (箇条書き `-` を推奨) を含めてください。

## デザイン・書式指定 (MARPディレクティブ含む)
- 全体として、シンプルで見やすいデザインを希望します。
- 可読性を最優先し、以下の点を考慮してください:
    - **余白**: スライド内の余白はやや狭めに設定してください。
    - **フォントサイズ**: 見出しは大きく、本文も読みやすい十分な大きさを確保してください。
    - **フォント**: 可能であれば `Noto Sans JP` を指定してください。
- 以下の基本的なMARPディレクティブを含めてください:
  \`\`\`marp
  ---
  marp: true
  theme: uncover
  paginate: true
  footer: '山梨大学 グローバル推進課 | [日付]' {/* フッターも修正 */ }
  header: '[プレゼンテーションタイトル]'
  style: |
    section {
      font-family: 'Noto Sans JP', sans-serif;
      padding: 40px;
    }
    h1, h2 {
      font-weight: bold;
    }
    p, ul, ol {
      font-size: 22px;
    }
  ---
  \`\`\`
  (※上記`style:`内のCSSは例です)

## その他
- 各スライドの内容について、補足説明やスピーカーノート案も可能であれば提案してください。

    ※MARP形式のコードブロック内にある ```marp ... ``` の部分は、HTML内で正しく表示するためにバッククォート(`)をエスケープしています（\`\`\`marp）。実際にコピーして使用する際は、元の3つのバッククォートに戻してください。
